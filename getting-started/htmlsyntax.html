HTML UI & CSS syntax

This page is about everything related to the app and its HTML side and settings. Involt converts CSS classes into specified user interface element, function and its parameters. The ard class defines the Involt object.

Everything must be included in index.html and created as single page layout (still you can add the pagination system similar to JQuery mobile or use Involt layout).

Element structure

The most basic explanation of syntax looks like this:

ard [element] [pin] [value] [parameters]

As a class within HTML element, it looks like this:

<div class="ard [element] [pin] [value] [parameters]"></div>

Example of rangeslider:

<div class="ard rangeslider P9 value-0 range-0-255 step-5"></div>

The list of elements and what parameters they need check reference page.

Involt pins

Pins are not only the direct representation of device pins (A0 analog pin as A0 pin in app). Keeping the Pin-to-Pin relation is most intuitive and highly recommended to maintain readability of your code. However, you can send more values than your device pin numbers so you can have control over some specific calculations. Moreover, if you don't wish to use the value for digital/analog write, for example - send string and show it on LCD display, you can also send that as Involt pin even if it's not used for any pin directly. It's possible to send values to things connected via expander with unused pins - for example Uno have 13 pins, so extend the pin index over 13 to distribute the data (as fake pins). Treat them as a container of values that by sending or receiving are accessible from both sides. Below there are explanations of pin types:

P - Send numeric value (only)

S - Send string, S0 is not the same as P0 and is send to different array in Arduino sketch.

A - Read only (string and numeric)

Value

It defines the value that element is going to send on specific action. It defines the beginning value of dynamic elements: like knob and rangeslider (default is 0). It can be numeric value or string (with S pin).

value-[value]

Some elements requires two values:

value-[value1]-[value2]

Shorthand for single numeric value, it must be right after the pin:

[pin] [number]

When string is longer than one word use string HTML attribute:

<div class="ard button S0" string="send something">Click me</div>

Parameters

The parameters determine how elements behave, you don't have to add them in specific order nor add them at all, if you use their default values.

Range - used to define minimum and maximum range of element. For read only default range is 0-1024 and for sending it's 0-255.

range-[minimum]-[maximum]

Step - the increment or decrement step of dynamic elements. Default is 1.

step-[value]

Function - send additional function name to device when element event occurs so the device can react to certain events in app. The function can be triggered immediately in Arduino sketch. Check the functions page.

fn-[name]

Editing the CSS

For customizing the visual aspects of your app, the most important file is framework.css - it contains styling of Involt elements. The layout-components.css have styling of loader and Involt pagination system. These stylesheets are not required if you don't use them so feel free to customize you layout. Keep in mind that some elements generates additional objects and you have to include thieir styling in your new stylesheet to make them work (things like positioning the handle in rangeslider).

Prototyping UI kit for wireframing

For the purpose of fast prototyping that Involt is all about you can uncomment the proto.css to make simplified "overlay" of each element to make it look like a wireframe. By working as UX designer you probably know how important it is.

App settings

How to change the project name and description?

On desktop, open manifest.json. Do not change other elements inside manifest because your project will disappear from launcher. For details about manifest file go to Chrome Manifest website. Some changes requires app to reload from chrome extensions menu.

On mobile, edit the config.xml

How to remove the loader?

  1. Open settings.js
  2. Change the loaderOnLaunch to false
  3. Set default connection settings below. After that Involt will connect to selected device on launch.
  4. Enjoy not selecting the device each time when app is launched

How to change the desktop window dimensions?

Open core/background.js and change screenWidth or screenHeight. To run app on fullscreen uncomment the fullscreen parameter.

Read-only elements update rate.

The update rate affects smoothness of updating HTML read elements. Default value is compromise between smoothness and cpu usage. Lower value will update objects almost in real time but will affect CPU usage (important especially on mobile). To change this open settings.js and change updateRate. This will not affect the connection bitrate.

Persistent connection

Due to many issues when working on code from both sides, Involt now has improved connection management. Previous versions blocked connected serial port and made tests frustrating. Now when connecting, user can decide to connect to port and closing previous connections or resume connection without rebooting device.

Chrome keeps the connection for very, very short time by default. This can be extended by changing isPersistent to true in settings.js. Keep in mind that with this mode it's impossible to upload sketch to device.

Issue affects only a serial connection, the bluetooth connection is not blocking the port when paired. Remember about disconnecting RX/TX pins during sketch upload.

Debug mode

Debug mode prints to console additional informations about what is happening in Involt and connection settings. To turn on debug mode open settings.js and change debugMode to true.

Involt is ready to use Chrome packaged app template. You can use it to quickly create interactive object with software to hardware communication. The goal is to help designers to overcome the obstacles at prototyping stage.
