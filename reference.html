Reference

Here you can find list of all HTML UI elements. Remember about writing involtSend() function for read-only elements and digitalWrite or analogWrite for data sending elements in Arduino sketch with corresponding involtPin[index].

The [value1-value2] mean that the value parameter requires two values value-[value1]-[value2]. [startvalue] is the beginning state of dynamic elements.

The value and parameters can be arranged in any order. Only the ard [element] [pin] must remain unchanged.

For JQuery methods reference check the JQuery page

Read data

Show

Display the analog pin value or received string.

ard show [pin]

<div class="ard show A0"></div>

Bar

Generates horizontal bar for value of analog pin. Width of bar value is automatically scaled to CSS width of element. For scaling the value use arduino map function. Add the label-constant class to show the value in constant position.

ard bar [pin] [range]

<div class="ard bar A0 range-0-1024"></div>

Knob

Generates knob that shows analog value. Based on JQuery Knob plugin by Anthony Terrien. You can add the data- attributes as described in plugin to change parameters of this element. For using proto UI kit add proto class to make it consistent with UI.

ard knob [pin] [range]

<div class="ard knob A0 range-0-1024"></div>

Value

Set html object value attribute to received data.

ard value [pin]

<div class="ard value A0"></div>

Send data

Button

The most basic UI element of Involt. Send the value (numeric or string) to target pin when button is pressed.

ard button [pin] [value]

<div class="ard button P5 value-100"></div>

Button-toggle

Toggle PWM pin between [value1] and [value2]. [value1] is also pin state on Arduino startup (should be declared in void setup( ) ).

ard button-toggle [pin] [value1-value2]

<div class="ard button-toggle P5 value-50-100"></div>

Toggle

Toggle digital pin ON and OFF. [startvalue] is pin state on Arduino startup (need to be declared in void setup() ). The pinMode is requied to correctly toggle between digital values. For switching between two PWM values use button-toggle.

ard toggle [pin] [startvalue]

<div class="ard toggle P5 value-0"></div>

Switch

Toggle between two values with mobile-like switch. To start with active state add active class.

ard switch [pin] [inactive-active]

<div class="ard switch P5 value-0-255"></div>

Increase

Increase pin value by [step]. [range] is limits of value. Works with decrease but both need to have same [startvalue]. You can add in void setup the [startvalue] (this is not required).

ard increase [pin] [startvalue] [range] [step]

<div class="ard increase P5 value-100 range-0-255 step-10"></div>

Decrease

Decrease pin value by [step]. [range] is limits of value. Works with increase but both need to have same [startvalue]. You can add in void setup the [startvalue] (this is not required).

ard decrease [pin] [startvalue] [range] [step]

<div class="ard decrease P5 value-100 range-0-255 step-10"></div>

Rangeslider

Generates JQuery based slider with noUiSlider plugin. The value is displayed inside a tooltip over object. Add the label-constant class to show the value in constant position. To access the JQuery object use .rangeslider>.slider selector, for more info read the plugin documentation. To send the values in real time, not only on release add the fluid class (may cause buffer overloads so step 1 is not recommended).

ard rangeslider [pin] [startvalue] [range] [step]

<div class="ard rangeslider P5 value-0 range-0-255 step-5"></div>;

Hover

Create element that sends value on hover and mouse leave. To combine hover with other Involt element you can simply use this as parent of target in html.

ard hover [pin] [onLeave-onHover]

<div class="ard hover P5 value-0-255"></div>

Knob-send

Simple knob that sends value. This knob is also based on JQuery Knob plugin - you can add plugin data- attributes to this element and they will be transfred to generated object. For using proto UI kit add proto class to make it consistent with UI. To send the values in real time, not only on release add the fluid class (may cause buffer overloads so step 1 is not recommended).

ard knob-send [pin] [startvalue] [range] [step]

<div class="ard knob-send P5 value-100 range-0-255 step-1"></div>

Custom-button

Create button that sends raw string data without target pin to device. Example code shows how to send PWM value 100 to pin 5. You can use it to send custom text which can be used for displaying on lcd display. You can do the same with JQuery .sendString() method.

ard custom-button [string]

<div class="ard custom-button P5V100"></div>

Involt-cta

Technically not a part of Involt UI kit but anything not from the reference list with that class will send a function. For sending functions with Involt UI element just use fn class. With involt-cta you have to add the function name as fn HTML attribute (not CSS).

For functions check the triggering functions tutorial.

This element will also work great with involt-pagelink in Involt pagination where going to another page will trigger function in device.

involt-cta (+functionName)

<div class="involt-cta" fn="functionName"></div>

HTML inputs

Involt-input

A general purpose class for adding hardware communication to non-Involt HTML inputs (rangesliders, text inputs, select etc.). For some elements like checkbox it's better to use Involt element. In more advanced project you should consider your own JQuery event.

ard involt-input [pin]

<input type="range" class="ard involt-input P10" min="0" max="100" value="50">

Input-write

Input which sends written value to chosen pin. You can use the type="number" to create input similar to increase/decrease element.

ard input-write [pin]

<input class="ard input-write P5">

Custom-write

Input which sends written text to device as string (non-direct only).

ard custom-write

<input class="ard custom-write">

Checkbox

Create checkbox with different values on [checked] and [unchecked] input state.

ard checkbox [pin] [unchecked-checked]

<input type="checkbox" class="ard checkbox P5 value-0-255">

Radio

Create radiobox with selected value. You can define same input name attribute to switch between values in group of radiobox objects.

ard radio [pin] [value]

<input type="radio" class="ard radio P5 value-0" name="myname">
<input type="radio" class="ard radio P5 value-100" name="myname">
<input type="radio" class="ard radio P5 value-255" name="myname">

Submit-button

Send values of HTML inputs as regular HTML form with submit button. This will prevent default sending event of html inputs (or rangeslider and knob-send) to trigger only when the button is pressed, passing all the values at once. The button and all target elements must be inside form tag. You can add the fn="functionNameString" HTML attribute to trigger function in arduino when pressed. Submit-button has no other parameters.

ard submit-button

<form>

<input type="text" class="ard input-write S0" placeholder="Name">

<input type="text" class="ard input-write S1" placeholder="Surename">


<button type="button" class="ard submit-button">Submit</button>

</form>
Involt is ready to use Chrome packaged app template. You can use it to quickly create interactive object with software to hardware communication. The goal is to help designers to overcome the obstacles at prototyping stage.
