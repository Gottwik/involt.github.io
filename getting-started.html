Getting Started - Quick start

Involt works like bridge between hardware and software. It's HTML/CSS framework bundled with hardware interactions - just define parameters of what to send where with CSS classes. The hardware part is reduced to minimum where framework handles only the communication and triggering functions. Involt from technical point of view does two things. It transfers variables between Arduino and app so they are accessible from both sides. Secondly, it contains UI kit generated by CSS classes and manipulated by JQuery.

The purpose of Involt is fast and easy prototyping - for more details about the project read the about page

What communication methods are supported?

  • Serial - desktop
  • Bluetooth (2.0/classic) - desktop and mobile
  • Bluetooth LE (early beta) - mobile

Even if it's HTML based, currently Involt has no IoT online support. However, for prototyping purpose it's rarely required to have this kind of communication or online service inluded at early stage.

How to install

Involt on desktop is created on top of Chrome API as offline Chrome Packaged App. It's easy to setup and suitable for Involt where Chrome dependency is not that important at all. The project works on Windows, OS X and Chrome OS (On Linux serial should work). For mobile installation go to Mobile page.

This guide uses serial connection by default.

  1. Install Google Chrome.
  2. Download and unpack Involt.
  3. In Chrome go to tools > extensions.
  4. Toggle Developer mode
  5. Click "Load unpacked extension..." and choose www folder.

Now open your project with launch button or Chrome App Launcher. If everything is ok you should see the window on the right. On the screen you should see available serial port lists and your device plugged in.

Adding UI element

To create Involt HTML element, describe its type and parameters as CSS classes. By doing this, the device is controlled without actually programming from app side. These parameters are consistent for each object so understanding how basic button works will similarly adapt to other UI elements. Below are two most basic examples. Read more: HTML UI & CSS syntax.

ard [element] [pin] [value] [parameters]

Read data from Arduino

Reading data from Arduino is easy. Everything you need is the involtSend(pinIndex, value) function inside included (involt-basic) sketch and HTML UI element you wish to communicate in index.html. Value of pin is stored and updated automatically from both sides.

Involt requires to write code from both sides, usually it's one line in app and one in sketch. The advantage of this solution is the fact that it's easier to adapt existing device and you can easily operate with basics of programming in Arduino IDE without any advanced topics.

It's highly recommended that you do most of the value calculations within your device. Otherwise you need to use in-app functions which is harder.

Add this to arduino sketch:

...

void loop(){

involtReceive();


involtSend(0, analogRead(A0));

delay(2);

}

...

Now add this element to index.html file. After connecting something to A0 (knob or anything else) you should see the value changing.

<div class="ard show A0"></div>

Send data to Arduino

Sending data requires digitalWrite or analogWrite with involtPin[pinIndex] array value. The values in this array are updated automatically. Sending and receiving can be related to any variable you want - not only pin states. You can use this array to store values for sketch functions.

...

void loop(){

involtReceive();


analogWrite(5, involtPin[5]);

}

...

The button will send pwm value 123 to pin 5 and turn on connected LED.

<div class="ard button P5 value-123">Click me</div>

Summary

As you see, using Involt is easy to understand. There is more you can do with Involt and it's widely explained in next chapters. Playing with examples is good idea to practice.

Sounds trivial? Maybe. Creating layouts with CSS sounds like something banal but have you ever thought about adding hardware interactions with it directly, without any online controled services or figuring out how to establish connection? With Involt, combining interactions is easy as never before and opens new ways in prototyping and further development. Calling this way of designing "tangible css" is something well describing what actually happens when you use Involt.

Depending on your workflow, whether its focused on layout, front end, or device itself, developing and iterating both sides of project at once is crucial and can't be separated. It's good to know how Involt works and what values must be transfered before starting the project (not everything must be send to app/device). It's easy to start from both ways - from app to device and vice versa. Thanks to CSS, another great thing in Involt is the ease with which you can hook up the hardware when you discovered Involt halfway through the project.

Involt is ready to use Chrome packaged app template. You can use it to quickly create interactive object with software to hardware communication. The goal is to help designers to overcome the obstacles at prototyping stage.
